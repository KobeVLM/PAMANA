## 3.4 Requirements Validation

This section describes how each category of requirements in this SRS will be verified and validated before and during the Capstone 1 MVP evaluation period.

| **Requirement Type**                                        | **Validation Method**                                                                                                                                                                                                                                                                                                                                                                          |
| ----------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Functional Requirements - all use cases (UC-SF.1 to UC-4.4) | Integration testing: each use case executed end-to-end in the deployed system. Pass criteria: all Normal Flow steps complete without system errors; all Alternative Flows trigger expected responses and error messages.                                                                                                                                                                       |
| Educational Content Alignment - MATATAG Q1-Q2               | Semi-structured interview with ≥2 Grade 2 Filipino teachers. Teachers review all 25 vocabulary words, syllable sets, and sentence tasks against the official MATATAG Q1-Q2 curriculum competencies using a content validation checklist. Pass criteria: ≥90% of all content items confirmed as curriculum-appropriate.                                                                         |
| NFR-P1 to NFR-P6 - Performance                              | Browser developer tools (Network and Performance panels) measure: audio playback start time (3 test runs per audio asset), visual feedback latency (3 test runs per interaction type), Dashboard load time, Klase leaderboard update latency, PDF generation time. Pass criteria: all measured values meet stated ≤ thresholds.                                                                |
| NFR-S1 - HTTPS                                              | Deploy to production (Vercel + Railway). Verify HTTPS certificate active in browser address bar. Attempt HTTP access and confirm automatic redirect to HTTPS. Pass criteria: redirect confirmed on Chrome, Firefox, Edge, Safari.                                                                                                                                                              |
| NFR-S2 - JWT Validation                                     | Postman: send API requests with (a) valid JWT, (b) expired JWT, (c) missing Authorization header. Pass criteria: (a) HTTP 200 with correct response, (b) HTTP 401, (c) HTTP 401.                                                                                                                                                                                                               |
| NFR-S3 - RLS Enforcement                                    | Supabase SQL editor: attempt cross-user SELECT queries simulating a logged-in user attempting to read another user's records. Pass criteria: all cross-user queries return 0 rows.                                                                                                                                                                                                             |
| NFR-R1 - Auto-save and Resume                               | Close browser mid-session immediately after completing a sub-level (not between sub-levels). Re-login and verify the system places the learner at the correct next position. Repeat 3 independent trials. Pass criteria: 3/3 correct resume positions verified.                                                                                                                                |
| NFR-U1 to NFR-U4 - Usability                                | System Usability Scale (SUS) questionnaire administered to 5 parents post-intervention. Target SUS score ≥70. Additionally, observe 3 Grade 2 pilot users completing Module 1 without any written instruction assistance and without adult verbal guidance. Pass criteria: all 3 pilot users complete Module 1 using audio cues only, with observer noting any confusion points for iteration. |
| API Correctness - all 21 endpoints                          | Postman collection: test all endpoints against expected request bodies, header requirements, and response schemas. Pass criteria: all endpoints return expected HTTP status codes and correct response JSON structure.                                                                                                                                                                         |
| Database Schema - RLS, FK, UNIQUE                           | Supabase dashboard + SQL editor: verify all RLS policies active, all foreign key constraints enforced, all UNIQUE constraints enforced via deliberate constraint violation tests. Pass criteria: all violations rejected at database level.                                                                                                                                                    |
| Browser Compatibility - NFR-C2                              | Manual walkthrough of Modules 1-4 complete game loops on Chrome 110+, Firefox 110+, Edge 110+, Safari 16+. Pass criteria: all game interactions function correctly on all 4 browsers with no visual layout breaks.                                                                                                                                                                             |
| Post-Intervention Curriculum Validation                     | Grade 2 Filipino teachers complete a post-intervention interview confirming that observed learner behavior in the system during the 4-week period aligned with expected MATATAG Q1-Q2 learning outcomes.                                                                                                                                                                                       |
